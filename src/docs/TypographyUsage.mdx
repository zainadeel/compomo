# Typography usage

Use `ds-text` for product copy. A typography variant is an atomic recipe:
font size, line height, weight, and letter spacing stay together.

## Choose a variant

| Variant | Typical use |
| --- | --- |
| `text-body-medium` | Default product copy and standard labels |
| `text-body-small` | Helper text, metadata, and supporting copy |
| `text-body-large` | Introductory copy or a modest hierarchy increase |
| `text-title-small` | Card titles and compact section headings |
| `text-title-medium` | Section headings and dialog titles |
| `text-title-large` | Page headings |
| `text-display-small` | Prominent summary values |
| `text-display-medium` | Rare hero metrics |
| `text-caption` | Compact labels and xs-density control text |

If no stronger hierarchy is required, start with `text-body-medium`.

```html
<ds-text variant="text-body-medium">
  Normal product copy
</ds-text>
```

## Semantics and appearance are separate

Use `as` for the correct native element and `variant` for the visual recipe.
A heading level describes document structure; it does not force a particular
visual size.

```html
<ds-text as="h1" variant="text-title-large">Page title</ds-text>
<ds-text as="h2" variant="text-title-medium">Section title</ds-text>
<ds-text as="p" variant="text-body-medium">Paragraph copy</ds-text>
<ds-text as="label" for="email" variant="text-body-small">Email</ds-text>
```

Available semantic elements are `p`, `span`, `div`, `label`, and `h1`–`h6`.

## Emphasis

`emphasis` is a boolean modifier, not a separate variant:

```html
<ds-text variant="text-body-medium" emphasis>
  Emphasized label
</ds-text>
```

Use emphasis to strengthen text within the same size tier. If the text is
actually a heading, choose the appropriate title variant instead.

Selected controls that use the shared interaction fill must not change text
weight dynamically. Selection should use the component's foreground and fill
contract while the base text recipe remains stable.

## Color hierarchy

Let text inherit the surrounding foreground by default. Use semantic `color`
values only when the text owns a clear hierarchy or intent:

```html
<ds-text color="secondary">Supporting copy</ds-text>
<ds-text color="negative">Validation error</ds-text>
```

Supported named values include `primary`, `secondary`, `tertiary`, `brand`,
`negative`, `positive`, `warning`, `caution`, `ai`, `on-strong`, `on-bold`,
and `inherit`. A TokoMo custom property such as
`var(--color-foreground-secondary)` is also accepted.

Do not use tertiary foreground for essential standard-size text without
checking contrast in its actual theme and surface context.

## Underlines and interaction

| Decoration | Meaning |
| --- | --- |
| `underline` | Text composed inside a real link |
| `dotted-underline` | Hidden or supplemental interaction such as a tooltip |

Decoration is visual only; it does not make `ds-text` interactive.

- A solid underline belongs inside a real `<a>` or another link component.
- A dotted underline requires an owning trigger that provides focus, keyboard
  interaction, and accessible tooltip or disclosure semantics.
- Do not alternate dotted and solid underlines to imply two competing actions.

## Wrapping and truncation

Use `wrap="nowrap"`, `wrap="balance"`, or `wrap="pretty"` when the text owner
has a clear layout reason. `lineTruncation` supports one through five lines:

```html
<ds-text variant="text-body-small" line-truncation="2">
  Long supporting copy
</ds-text>
```

The host needs a bounded width for truncation to occur. `ds-text` owns the
measurable text box, so apply width, flex, grid, and truncation classes directly
to it rather than adding a neutral wrapper.

## Numeric content

Use `font-feature="tabular-nums"` for columns, counters, or changing values
whose digits need stable widths.

## Avoid

- Raw typography utility classes in components when `ds-text` can express the
  recipe.
- Overriding a variant's font size, line height, weight, or letter spacing.
- Using display variants for ordinary headings.
- Using title variants for paragraph copy.
- Using underline styles as decoration.

See **Foundation → Typography** and **Primitives → Text** for the complete
visual and component references.
